Detailed Lab Modules

Each module mirrors the TryHackMe-style learning path with objectives, visuals, and core skills.

Module 01 Est. time: 10 minutes

This webpage is an interactive, guided phishing simulation lab‚Äîspecifically, a hands-on training module focused on dissecting a Clone Phishing attack.

It's designed as a self-paced exercise to elevate a user's security awareness, moving beyond simple definitions to the actual analysis of a threat. The lab meticulously walks the participant through the lifecycle of a credential harvesting attempt, structured across three key phases:

  • Scenario and Social Engineering Analysis The module opens with an incident brief centered on an employee, "Taylor," targeted by an email impersonating HR Benefits. The core of the attack lies in the social engineering tactics:
  • Urgency: It applies immediate time pressure by demanding a log-in "within 30 minutes or lose coverage," overriding the target's impulse to verify.
  • Typosquatting: The sender address employs a subtle but critical domain swap (peakbeneflts.com), designed to look legitimate at a glance, a classic sign of an attacker-controlled identity.
  • Forced Action: The message provides a single call-to-action link that, upon hovering, reveals the attacker's domain (security-update.io).
Skills: Email forensics ¬∑ Header review ¬∑ Threat awareness
Module 02 Est. time: 15 minutes

Technical and Visual Inspection

Next, the lab shifts focus to the cloned portal itself. This section trains the user to spot both high-level visual cues and underlying technical failures that expose the fraud:

  • Domain & Certificate Discrepancies: The mock portal uses a suspicious, unbranded URL (security-update.io) and lacks a trusted SSL certificate, a tell-tale sign of a quick-setup phishing kit.
  • Branding Inconsistencies: Outdated logos, mismatched color palettes, and irrelevant copyright years signal that the site is a hastily assembled replica, not the legitimate corporate asset.
  • Credential Harvesting Mechanism: Crucially, the lab highlights that any submitted credentials are sent unencrypted over plain HTTP to a collection API, and that the portal dangerously asks for a Multi-Factor Authentication (MFA) code on the same page. This prepares the participant for the concept of MFA Relay Attacks, where an attacker captures and immediately replays the one-time code to bypass security.
  • Email Header Forensics: It also includes a technical detail from the email header, showing spf=fail, which is a definitive indicator that the sending server was not authorized to send mail on behalf of the claimed domain, thus confirming the email spoofing.
Skills: Web cloning analysis ¬∑ Credential defense ¬∑ UX anomaly spotting
